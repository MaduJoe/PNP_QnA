# 🔥 PNP QnA Bot - 하이브리드 검색 시스템

## 📋 문제 해결

### ❌ **기존 문제점**
사용자가 지적한 문제:
- **"매니저 로그인 관련 문의"** 검색 시
- Top 1 결과가 **"[공유] DBSAFER IM 사용 시 SU 계정을 통한 자동 접속 기능"**
- **질의한 키워드가 결과에 포함되지 않는 문제**
- **임베딩만으로는 키워드 검색보다 신뢰성 부족**

### ✅ **해결 방안: 하이브리드 검색**

**키워드 검색 + 임베딩 검색 결합**으로 정확도 대폭 향상!

## 🛠️ **하이브리드 검색 시스템 구조**

### 🔍 **검색 엔진 구성**

1. **TF-IDF 키워드 검색** (가중치: 60%)
   - 정확한 키워드 매칭
   - 제목에 5배 가중치 부여
   - 한국어 최적화 (1-2gram, word 단위)

2. **의미 임베딩 검색** (가중치: 40%)
   - 의미적 유사도 계산
   - 한국어 특화 모델 (`jhgan/ko-sroberta-multitask`)

3. **정확한 매칭 보너스** (최대 70% 추가)
   - 제목 매칭: 50% 보너스
   - 내용 매칭: 20% 보너스

### 📊 **점수 계산 공식**

```
최종점수 = (키워드점수 × 0.6) + (임베딩점수 × 0.4) + 매칭보너스
```

## 🔧 **기술적 세부사항**

### **TF-IDF 설정**
```python
TfidfVectorizer(
    max_features=10000,
    ngram_range=(1, 2),      # 1-2gram 사용
    analyzer='word',         # 단어 단위 분석
    sublinear_tf=True        # 로그 스케일링
)
```

### **임베딩 모델**
- **모델**: `jhgan/ko-sroberta-multitask`
- **특징**: 한국어 특화, 다중 태스크 학습
- **차원**: 768차원 벡터

### **가중치 조정**
```python
keyword_weight = 0.6    # 키워드 검색 비중 증가
embedding_weight = 0.4  # 임베딩 검색 비중 감소
```

## 🎛️ **하이브리드 검색 설정**

`config.yaml`에서 가중치 조정 가능:

```yaml

  keyword_weight: 0.6      # 키워드 검색 가중치
  embedding_weight: 0.4    # 임베딩 검색 가중치

  exact_match_bonus: 0.7   # 정확한 매칭 보너스
```
